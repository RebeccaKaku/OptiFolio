# FM 金融管理系统 - 快速开发指南

## 🎯 目标读者
本文档适合**新加入的开发者**快速上手FM系统，用最简单的方式理解系统并开始开发。

## 🚀 5分钟快速上手

### 第1步：运行系统

## 🔧 如何添加新功能

### 场景1：我要添加一个新的资产类型

**例子**：添加"加密货币"支持

**步骤**：
1. 创建数据获取器：`src/data_core/fetchers/crypto_fetcher.py`
2. 复制现有fetcher的模板，修改数据源
3. 在`src/core/asset_manager.py`中注册新类型

**代码模板**：
```python
# src/data_core/fetchers/crypto_fetcher.py
import yfinance as yf
import pandas as pd

class CryptoFetcher:
    def fetch(self, symbol: str) -> pd.DataFrame:
        # 获取比特币数据，例如：BTC-USD
        ticker = yf.Ticker(f"{symbol}-USD")
        return ticker.history(period="1y")
```

### 场景2：我要添加一个新的API接口

**例子**：添加"获取资产新闻"功能

**步骤**：
1. 在`src/api/`目录创建新文件：`news_api.py`
2. 复制现有API的模板结构
3. 在`src/api/api_service.py`中集成

**代码模板**：
```python
# src/api/news_api.py
class NewsAPI:
    def get_asset_news(self, symbol: str):
        # 实现获取新闻的逻辑
        return {"news": ["新闻1", "新闻2"]}
```

### 场景3：我要修改现有功能

**原则**：找到对应的文件，小步修改，充分测试。

**例子**：修改资产导入的货币识别逻辑
- 找到文件：`src/asset_importer.py`
- 找到方法：`_fetch_currency_info`
- 修改逻辑，测试验证

## 🐛 常见问题解决

### 问题1：导入资产失败
**检查清单**：
- ✅ 网络连接是否正常？
- ✅ 资产代码是否正确？
- ✅ 资产类型是否匹配？
- ✅ 查看错误信息，通常是数据源问题

### 问题2：系统启动失败
**检查清单**：
- ✅ Python版本 >= 3.8？
- ✅ 依赖包是否安装完整？
- ✅ 配置文件是否存在？
- ✅ 端口8501是否被占用？

### 问题3：数据获取失败
**检查清单**：
- ✅ akshare/yfinance是否正常工作？
- ✅ 防火墙是否阻止了外部API？
- ✅ 是否有API访问限制？

## 🎯 开发最佳实践（记住这5条）

1. **先跑起来，再优化** - 不要一开始就追求完美
2. **小步快跑，频繁测试** - 每次修改都要测试
3. **复制粘贴是好朋友** - 从现有代码开始修改
4. **错误信息要友好** - 用户看不懂技术错误
5. **文档跟着代码走** - 改了代码就要更新文档
