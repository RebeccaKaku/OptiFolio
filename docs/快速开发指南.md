## 5分钟快速上手

### 第1步：启动 API 服务

```powershell
# 激活环境
conda activate optifolio313

# 启动 FastAPI 服务（端口 8011）
python tools/start_app.py
```

浏览器打开 `http://localhost:8011/health` 看到 `{"status": "ok"}` 即成功。

> **注意**：旧的 `streamlit run app.py` 已冻结，不再用于新功能开发。所有新接口在 FastAPI。

### 第2步：获取市场数据

```powershell
# 从 FinData 拉取资产价格并存入规范存储
python tools/ingest_portfolio_prices.py

# 或者直接用 API
curl "http://localhost:8011/api/market/prices?assets=AAPL,QQQ"
```

### 第3步：查看投资组合

```powershell
# 当前净值
curl "http://localhost:8011/api/portfolio/v2/value?as_of=$(date -I)"

# 风险分析
curl "http://localhost:8011/api/portfolio/v2/risk/liquidity"
curl "http://localhost:8011/api/portfolio/v2/risk/concentration"
```

## 系统架构（一句话理解）

```
数据源 → FinData适配器 → 质量检查 → 规范存储 → 服务层 → FastAPI → 前端
```

**核心原则**：FinData 是唯一的数据入口。所有市场数据通过 `from FinData import fd` 访问，禁止直接导入 fetcher。

## 文件结构（记住这6个目录）

```
OptiFolio/
├── FinData/              # 数据中心（自包含，32个文件）
│   ├── adapters/         # 数据源适配器（美股/港股/基金/外汇/银行理财）
│   ├── store/            # 存储引擎（QualityGate 8项检查）
│   ├── orchestration/    # 调度/频率控制/回退链
│   └── serving/          # 公共数据API
├── src/
│   ├── api/              # FastAPI 路由（端口8011）
│   ├── services/         # 业务服务层
│   ├── analytics/        # 分析引擎（告警/敞口/集中度/流动性/外汇）
│   ├── core/             # 核心（估值/日历/公司行动/费率）
│   └── data_foundation/  # 规范数据格式定义
├── config/               # YAML 配置文件
├── tools/                # 命令行工具
└── tests/                # 31个测试文件，592个用例
```

> `app.py` 是遗留的 Streamlit 仪表盘（1550行），已冻结，**不要在上面添加功能**。

## 如何添加新功能

### 场景1：添加新的数据源

**步骤**：
1. 在 `FinData/adapters/` 创建新文件（如 `hk_stock.py`）
2. 实现 `FetcherProtocol` 接口，返回 `FetchResult`
3. 在 `FinData/adapters/__init__.py` 的 `FETCHER_REGISTRY` 注册
4. 在 `FinData/orchestration/cadence.py` 配置更新频率

```python
# FinData/adapters/hk_stock.py
from FinData.adapters import FetcherProtocol, FetchResult

class HkStockFetcher(FetcherProtocol):
    def fetch(self, symbol, start_date, end_date, **kwargs) -> FetchResult:
        # 获取港股数据
        ...
        return FetchResult(symbol=symbol, dataframe=df, success=True)
```

### 场景2：添加新的 API 接口

**步骤**：
1. 如果需要新业务逻辑，在 `src/services/` 添加方法
2. 在 `src/api/fastapi_app.py` 添加路由

```python
# src/api/fastapi_app.py
@app.get("/api/my-new-endpoint", tags=["my-feature"])
def my_endpoint(symbol: str) -> JSONResponse:
    return _json_response(
        get_application_services().my_service.do_thing(symbol)
    )
```

### 场景3：添加新的分析/告警

**步骤**：
1. 在 `src/analytics/` 创建分析模块
2. 在 `src/analytics/alerts.py` 的 `AlertEngine` 中添加检查方法
3. 检查方法签名：`def check_xxx(self, ...) -> Optional[Alert]`

## FinData 快速使用

```python
from FinData import fd

# 价格
aapl = fd.prices("AAPL")                    # Series
panel = fd.panel(["AAPL", "QQQ"])           # DataFrame (date × asset)

# OHLCV
ohlcv = fd.ohlcv("AAPL")                    # DataFrame (open/high/low/close/adj_close/volume)

# 收益
returns = fd.returns("AAPL")                # Series (日收益)

# 指标
metrics = fd.metrics("AAPL")                # dict (sharpe, volatility, max_drawdown, ...)

# 汇率
rate = fd.fx_rate("USD", "CNY")             # float

# 导出
fd.export(["AAPL", "QQQ"], "csv", "prices.csv")
```

## 常见问题

### 问题1：启动 API 失败
- ✅ Python 版本 ≥ 3.11？
- ✅ `conda activate optifolio313` 已执行？
- ✅ `config/secrets.yaml` 存在？（复制 `config/secrets.example.yaml`）
- ✅ 端口 8011 未被占用？

### 问题2：数据获取失败
- ✅ 网络连接正常？
- ✅ `FinData/orchestration/ingest.py` 中资产代码正确？
- ✅ 查看 `FinData/store/quality.py` 的 QualityReport

### 问题3：测试失败
- ✅ 使用 `--basetemp .pytest_tmp -p no:cacheprovider` 参数？
- ✅ `local/portfolio.yaml` 存在？（某些测试需要）
