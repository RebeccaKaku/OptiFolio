## 代码架构理解

### 1. 分层架构

FM系统采用清晰的分层架构，每层有明确的职责：

```
UI层 (Presentation)
    ↓
API层 (Interface) 
    ↓
业务逻辑层 (Business Logic)
    ↓
数据访问层 (Data Access)
    ↓
数据源层 (Data Sources)
```

### 2. 核心模块职责

| 模块 | 位置 | 主要职责 |
|------|------|----------|
| **资产管理** | `src/core/asset_manager.py` | 资产生命周期管理，类型注册 |
| **组合管理** | `src/core/portfolio_core.py` | 持仓管理，价值计算，风险分析 |
| **仪表板引擎** | `src/core/dashboard_engine.py` | 数据聚合，图表生成，指标计算 |
| **API服务** | `src/api/api_service.py` | 统一API接口，错误处理 |
| **数据Fetchers** | `src/data_core/fetchers/` | 工厂模式数据获取，支持多种资产类型 |
| **数学引擎** | `src/math_engine.py` | 金融数学计算，统计量计算 |
| **策略引擎** | `src/strategy.py` | 投资策略，优化算法 |

### 3. 数据流示例

以下是一个典型的资产导入数据流：


### 3. 错误处理规范

#### 异常类定义
```python
# src/core/exceptions.py
class FMError(Exception):
    """FM系统基础异常类"""
    def __init__(self, message: str, error_code: str, details: Dict = None):
        self.message = message
        self.error_code = error_code
        self.details = details or {}
        super().__init__(self.message)

class AssetNotFoundError(FMError):
    """资产未找到异常"""
    pass

class DataValidationError(FMError):
    """数据验证失败异常"""
    pass
```

#### 错误处理模式
```python
from src.core.exceptions import AssetNotFoundError, DataValidationError

def get_asset_price(symbol: str, date: str) -> float:
    """获取资产价格（包含错误处理）"""
    try:
        # 尝试获取数据
        price_data = fetch_price_data(symbol, date)
        
        # 数据验证
        if not price_data:
            raise AssetNotFoundError(
                message=f"资产 {symbol} 在 {date} 的价格数据未找到",
                error_code="ASSET_PRICE_NOT_FOUND",
                details={"symbol": symbol, "date": date}
            )
            
        if price_data <= 0:
            raise DataValidationError(
                message=f"资产 {symbol} 的价格数据无效",
                error_code="INVALID_PRICE_DATA",
                details={"symbol": symbol, "price": price_data}
            )
            
        return price_data
        
    except AssetNotFoundError:
        # 重新抛出业务异常
        raise
        
    except Exception as e:
        # 包装系统异常
        raise FMError(
            message="获取资产价格时发生系统错误",
            error_code="PRICE_FETCH_SYSTEM_ERROR",
            details={"symbol": symbol, "original_error": str(e)}
        )
```

```python
# src/core/metrics/custom_metrics.py
import numpy as np
from typing import List, Dict, Any

def calculate_custom_metric(
    prices: pd.DataFrame,
    period: int = 20
) -> Dict[str, Any]:
    """计算自定义指标
    
    Args:
        prices: 价格DataFrame，包含'Close'列
        period: 计算周期
        
    Returns:
        包含指标值的字典
    """
    if len(prices) < period:
        return {"value": None, "error": "数据长度不足"}
    
    close_prices = prices['Close'].values
    
    # 计算指标逻辑
    # 例如：计算价格波动率
    returns = np.diff(np.log(close_prices))
    volatility = np.std(returns) * np.sqrt(252)  # 年化波动率
    
    return {
        "value": volatility,
        "period": period,
        "description": f"{period}日年化波动率",
        "format": ".2%"
    }
```

```python
    """批量获取资产信息（带缓存）"""
    results = []
    
    for symbol in symbols:
        # 尝试从缓存获取

        if cached:
            results.append(cached)
        else:

            cache_manager.set("asset", symbol, data, ttl=1800)
            results.append(data)
    
    return results
```

```python
    overall_status = "healthy" if all(checks.values()) else "unhealthy"
    
    return {
        "status": overall_status,
        "timestamp": datetime.now().isoformat(),
        "version": get_version(),
        "checks": checks
    }
```

```python
def import_asset_with_metrics(symbol: str, asset_type: str):
    """带监控指标的资产导入"""
    start_time = time.time()
    
    try:
        result = import_asset(symbol, asset_type)
        
        # 记录成功指标
        ASSET_IMPORT_COUNTER.labels(
            asset_type=asset_type,
            status='success'
        ).inc()
        
        ASSET_IMPORT_DURATION.labels(

        return result

        ASSET_IMPORT_COUNTER.labels(
            asset_type=asset_type,
            status='failure'
        ).inc()
        
        raise
```

## 贡献指南

### 1. 代码提交规范

#### 提交信息格式
```
类型(范围): 简要描述

详细描述（可选）

关闭的Issue（可选）：#123
```

#### 类型说明
- `feat`: 新功能
- `fix`: 修复bug
- `docs`: 文档更新
- `style`: 代码格式调整
- `refactor`: 代码重构
- `test`: 测试相关
- `chore`: 构建过程或辅助工具变动

#### 示例
```
feat(asset): 添加加密货币资产类型支持

- 添加 BitcoinFetcher 类
- 注册 'crypto' 资产类型
- 更新配置文件支持加密货币

关闭：#45
```

### 2. 代码审查流程

1. **创建分支**: 从`develop`分支创建功能分支
2. **开发功能**: 实现功能并编写测试
3. **本地测试**: 运行所有测试确保通过
4. **提交PR**: 创建Pull Request到`develop`分支
5. **代码审查**: 至少需要1位核心开发者审查
6. **CI检查**: 确保CI流水线通过
7. **合并**: 审查通过后合并到`develop`

### 3. 版本发布流程

1. **功能冻结**: 确定版本功能范围
2. **测试阶段**: 进行全面测试
3. **文档更新**: 更新相关文档
4. **版本标签**: 打上版本标签（如v1.2.0）
5. **发布公告**: 撰写发布说明
6. **部署**: 部署到生产环境
