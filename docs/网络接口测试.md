# 网络接口测试文档

## 概述

网络接口测试模块是OptiFolio的重要组成部分，用于测试各种金融数据API的连通性和响应状态。该模块可以帮助用户快速识别网络问题，确保系统能够正常获取金融数据。

### 主要特性

1. **多API测试** - 支持测试akshare、yfinance、外部网站等多种API接口
2. **异步并行测试** - 提高测试效率，减少等待时间
3. **详细测试报告** - 提供成功率、响应时间等关键指标
4. **缓存机制** - 支持测试结果缓存，避免重复测试
5. **导出功能** - 支持将测试结果导出为CSV格式
6. **集成到仪表板** - 可通过Web界面一键测试API状态
7. **货币基金专项测试** - 区分货币基金和普通基金的接口测试

## 测试内容

### 支持的API类型

1. **AkShare函数接口**
   - `stock_zh_a_spot_em` - A股实时行情
   - `stock_zh_a_hist` - A股历史行情
   - `fund_em` - 基金数据（普通基金）
   - `fund_money_rate_em` - 货币基金数据

2. **yfinance接口**
   - `Ticker` - 股票信息获取
   - `download` - 股票历史数据下载

3. **外部HTTP接口**
   - 新浪财经
   - 东方财富
   - 腾讯财经

4. **股票代码可用性测试**
   - 中国股票：sh600519、sz000001、510300、005827
   - 美股：AAPL、GOOGL、MSFT、TSLA
   - 基金：005827、002892、510300、000198

### 货币基金 vs 普通基金测试

系统专门区分了货币基金和普通基金的接口测试：

- **普通基金** (`fund_em`): 测试天天基金网的普通基金数据
- **货币基金** (`fund_money_rate_em`): 测试货币基金的万份收益和7日年化收益率

这种区分确保了系统能够正确识别和处理不同类型的基金数据。

## 数据获取模式分析

### 当前系统的数据获取策略

基于对代码的分析，OptiFolio采用以下数据获取模式：

#### 1. **全量获取模式** (Full Refresh)
- **适用场景**: 首次导入资产、用户手动刷新、定期数据更新
- **实现方式**: 每次获取完整的日期范围数据
- **优点**: 简单可靠，确保数据完整性
- **缺点**: 网络开销大，处理时间长

#### 2. **按资产类型区分的获取策略**

**中国股票/基金 (AkShare)**:
```python
# 获取完整历史数据
df = ak.stock_zh_a_hist(
    symbol="600519",
    period="daily", 
    start_date="20240101",
    end_date="20240102"
)
```

**美股 (yfinance)**:
```python
# 获取指定日期范围的完整数据
df = ticker.history(
    start=start_date, 
    end=end_date, 
    interval=interval,
    auto_adjust=adjust_price
)
```

**货币基金特殊处理**:
- 使用累计收益计算净值
- 从1.0开始，根据万份收益累计计算

#### 3. **存储和缓存策略**

**原始数据存储**:
- 按频率分类存储 (daily, minute, weekly, monthly)
- 支持Parquet和CSV两种格式
- 元数据存储在DataFrame属性中

**处理后数据存储**:
- 统一存储为market_matrix.parquet
- 包含所有资产的对齐数据

### 增量获取的潜在改进

#### 当前挑战
1. **数据源限制**: AkShare和yfinance接口不支持增量参数
2. **数据一致性**: 需要确保新旧数据的连续性
3. **时区处理**: 不同数据源的时区差异

#### 建议的增量获取方案

```python
class IncrementalDataFetcher:
    """增量数据获取器"""
    
    def get_incremental_data(self, symbol: str, last_date: str) -> pd.DataFrame:
        """获取自last_date以来的增量数据"""
        
        # 1. 确定增量范围
        start_date = pd.to_datetime(last_date) + pd.Timedelta(days=1)
        end_date = pd.Timestamp.now().normalize()
        
        if start_date >= end_date:
            return pd.DataFrame()  # 无新数据
        
        # 2. 获取增量数据
        new_data = self.fetch(symbol, start_date.strftime('%Y-%m-%d'), 
                            end_date.strftime('%Y-%m-%d'))
        
        # 3. 数据验证
        if not new_data.empty:
            # 确保数据连续性
            expected_start = start_date
            actual_start = new_data.index.min()
            
            if abs((actual_start - expected_start).days) > 1:
                print(f"警告: {symbol} 数据不连续，预期从{expected_start}开始，实际从{actual_start}开始")
        
        return new_data
    
    def merge_incremental_data(self, existing_data: pd.DataFrame, 
                              new_data: pd.DataFrame) -> pd.DataFrame:
        """合并增量数据到现有数据"""
        
        if new_data.empty:
            return existing_data
        
        if existing_data.empty:
            return new_data
        
        # 去重合并
        combined = pd.concat([existing_data, new_data])
        combined = combined[~combined.index.duplicated(keep='last')]
        combined = combined.sort_index()
        
        return combined
```

## 故障排除

### 常见问题

1. **测试超时**
   - 增加超时时间：`NetworkTester(timeout=30)`
   - 检查网络连接状态
   - 确认防火墙设置

2. **AkShare接口失败**
   - 检查akshare版本：`pip show akshare`
   - 确认网络是否能访问东方财富等数据源
   - 查看具体错误信息

3. **yfinance接口失败**
   - 检查yfinance版本：`pip show yfinance`
   - 确认能访问Yahoo Finance
   - 检查是否被防火墙阻止

4. **HTTP接口失败**
   - 检查是否能正常访问目标网站
   - 确认没有网络代理问题
   - 尝试直接访问URL测试

## 监控和告警

### 1. 定期监控

```python
import schedule
import time
from src.api.network_tester import run_network_test

def monitor_network():
    """定期监控网络状态"""
    report = run_network_test()
    success_rate = report['summary']['success_rate']
    
    if success_rate < 80:
        print(f"⚠️ 警告：网络成功率低于80% ({success_rate}%)")
        # 发送告警通知
    elif success_rate < 60:
        print(f"🚨 严重：网络成功率低于60% ({success_rate}%)")
        # 发送严重告警
    
    print(f"网络监控完成：成功率 {success_rate}%")

# 每30分钟检查一次
schedule.every(30).minutes.do(monitor_network)

while True:
    schedule.run_pending()
    time.sleep(1)
```

## 最佳实践

1. **定期测试**: 建议每小时或每天运行一次网络测试
2. **监控关键API**: 重点关注akshare和yfinance相关接口的状态
3. **设置告警阈值**: 成功率低于80%时发出警告
4. **缓存结果**: 避免频繁测试，合理使用缓存
5. **日志记录**: 保留测试历史记录，便于分析问题
6. **故障预案**: 准备备用数据源或降级方案

## 数据获取模式总结

### 当前模式
- **全量获取**: 每次获取完整日期范围数据
- **按类型区分**: 不同资产类型使用不同获取策略
- **智能降级**: 货币基金使用累计收益计算

### 增量获取建议
虽然当前系统使用全量获取模式，但可以通过以下方式实现增量更新：

1. **记录最后更新日期**: 在元数据中保存最后获取日期
2. **日期范围计算**: 只获取新数据
3. **数据合并**: 将新数据与现有数据合并
4. **验证连续性**: 确保数据没有缺口

这种混合模式可以在保证数据完整性的同时，显著提高获取效率。
