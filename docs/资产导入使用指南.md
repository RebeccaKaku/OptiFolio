# 资产导入使用指南

> ⚠️ 历史文档 — 2026年5月。当前架构请参阅 docs/AI_CONTEXT.md

## 概述

重构后的资产导入系统基于**显式配置**原则，解决了原有系统中的代码冲突问题，支持美元计价基金自动识别，提供了更强大的资产管理能力。

### 主要特性

1. **强制显式指定资产类型** - 无自动猜测，避免歧义
2. **支持代码冲突** - 同一代码可对应多个不同资产
3. **美元计价基金识别** - 自动从基金名称识别币种
4. **YAML配置管理** - 所有资产配置保存在文件中
5. **完整的API支持** - 自动从akshare/yfinance获取资产信息

## 处理代码冲突

重构系统专门解决了类似"000001"这样的代码冲突问题：

### 示例：000001既是股票又是基金

```python
from src.asset_importer import AssetImporter

importer = AssetImporter()

# 导入000001作为平安银行（深圳股票）
asset1 = importer.import_asset(
    '000001', 
    'cn_stock_sz', 
    conflict=True,  # 标记为冲突资产
    refresh=True
)

# 导入000001作为华夏成长混合（公募基金）
asset2 = importer.import_asset(
    '000001', 
    'cn_fund_open', 
    conflict=True,  # 标记为冲突资产
    refresh=True
)

print(f"资产1: {asset1.name} ({asset1.asset_type})")
print(f"资产2: {asset2.name} ({asset2.asset_type})")
print(f"冲突ID: {asset1.conflict_id}, {asset2.conflict_id}")
```

### 通过冲突ID获取特定资产

```python
from src.asset_importer import get_asset

# 获取000001的第一个冲突资产（平安银行）
asset1 = get_asset('000001', '000001_1')

# 获取000001的第二个冲突资产（华夏成长混合）
asset2 = get_asset('000001', '000001_2')
```

## 美元计价基金识别

系统自动从基金名称中识别币种：

### 支持的美元关键词

- 美元现汇
- 美元现钞
- 美元
- USD

### 示例

```python
from src.asset_importer import AssetImporter

importer = AssetImporter()

# 这些基金会被自动识别为USD
usd_funds = [
    ('002892', 'cn_fund_qdii'),  # 华夏移动互联混合(QDII)美元现汇
    ('008285', 'cn_fund_qdii'),  # 易方达全球医药行业混合发起式(QDII)A(美元现汇)
    ('000075', 'cn_fund_qdii'),  # 华夏恒生ETF联接A(美元汇)
]

for symbol, asset_type in usd_funds:
    asset = importer.import_asset(symbol, asset_type, refresh=True)
    print(f"{symbol}: {asset.name} -> {asset.currency}")
```

## 资产类型参考

### 支持的资产类型

| 资产类型 | 说明 | 示例代码 | 默认币种 |
|---------|------|----------|----------|
| `cn_stock_sh` | 上海A股 | 600, 601, 603, 605, 688 | CNY |
| `cn_stock_sz` | 深圳A股 | 000, 001, 002, 003, 300 | CNY |
| `cn_fund_open` | 场外公募基金 | 6位数字代码 | CNY |
| `cn_fund_etf` | 场内ETF基金 | 6位数字代码 | CNY |
| `cn_fund_qdii` | QDII基金 | 6位数字代码 | CNY（可能被覆盖为USD） |
| `cn_fund_money` | 货币基金 | 6位数字代码 | CNY |
| `cn_fund_lof` | LOF基金 | 6位数字代码 | CNY |
| `cn_fund_index` | 指数基金 | 6位数字代码 | CNY |
| `us_equity` | 美股 | 股票代码 | USD |
| `hk_stock` | 港股 | 股票代码 | HKD |
| `currency` | 货币 | 货币对代码 | USD |
| `cn_index` | 指数 | 指数代码 | CNY |

### 向后兼容的类型

| 旧类型 | 新类型 | 说明 |
|--------|--------|------|
| `cn_stock` | `cn_stock_sh` 或 `cn_stock_sz` | 根据代码自动判断 |
| `cn_fund` | `cn_fund_open` | 场外公募基金 |
| `cn_etf` | `cn_fund_etf` | 场内ETF基金 |

## 故障排除

### 常见问题

1. **资产导入失败**
   - 检查资产类型是否正确
   - 确保akshare/yfinance已安装
   - 检查网络连接

2. **币种识别错误**
   - 检查基金名称是否包含美元关键词
   - 可手动指定currency参数

3. **fetcher找不到**
   - 确保已运行 `patch_fetcher_factory()`
   - 检查资产类型是否已注册

4. **配置保存失败**
   - 检查config目录权限
   - 确保YAML格式正确

## 最佳实践

1. **始终显式指定资产类型**：避免使用自动检测
2. **处理冲突代码**：使用 `conflict=True` 参数
3. **定期更新配置**：使用 `refresh=True` 更新资产信息
4. **备份配置文件**：定期备份 `config/asset_registry.yaml`
5. **使用版本控制**：将配置文件纳入版本控制
6. **定期测试网络接口**：使用网络测试功能确保API可用性
