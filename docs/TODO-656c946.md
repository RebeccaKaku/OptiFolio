# TODO — Active Priorities

**Baseline:** OptiFolio 656 passed; FinDataProvider 191 passed before merge.

## P0 — Make FinDataProvider independently operable

1. **Autonomous scheduling.** The worker currently consumes queued query misses,
   but does not use `FINDATA_DAILY_SYNC_HOUR_UTC` to schedule periodic price,
   FX, macro-observation, or quality refreshes.
2. **Job reliability.** Add bounded retries, exponential backoff, dead-letter
   state, and failure visibility. A failed fetch is currently terminal.
3. **Production deployment.** Before moving off local WSL, define encrypted
   transport/private networking, token rotation, least-privilege service users,
   persistent-volume backup, and a tested restore procedure.
4. **Service observability.** Readiness should report queue depth, last
   successful refresh by dataset, failure counts, and data freshness—not only a
   worker heartbeat.

## P1 — Protect and speed up the service boundary

1. Add CI to both repositories: unit tests, boundary tests, Docker build, and
   privacy/secret scanning.
2. Reduce provider latency. The real book summary is about 9 seconds warm and
   about 20 seconds cold; profile Parquet scans, repeated FX requests, and
   provider initialization before adding application-side caches.
3. Make stale checks exchange-calendar aware. The current seven-calendar-day
   threshold is a conservative workaround, not a trading-calendar rule.
4. Enrich `/v1/assets` with public metadata or add a batch metadata endpoint so
   the dashboard does not render name/type/currency/source as `-`.
5. Finish coverage for products that remain unpriced, especially bank WMPs, and
   expose pending/failed ingestion state in the UI.

## P1 — Preserve the user journey

Turn the verified manual flow into a repeatable acceptance test:

```text
create account -> create product -> confirm snapshot -> remote valuation
-> My Money summary -> risk/exposure views
```
