# OptiFolio 架构设计文档

## 系统概述

OptiFolio是一个功能完整的多资产金融管理系统，提供资产导入、组合管理、风险分析和可视化监控等全方位功能。系统采用分层架构设计，支持高扩展性和灵活性。

## 整体架构

### 分层架构模式

```
┌─────────────────────────────────────────────────────────┐
│                     UI层 (Presentation)                   │
│                   Streamlit Web界面                      │
└─────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────┐
│                    API层 (Interface)                     │
│           统一API服务 + 模块化API接口                    │
└─────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────┐
│                  业务逻辑层 (Business Logic)              │
│    资产管理器 + 组合管理核心 + 仪表板引擎                │
└─────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────┐
│                  数据访问层 (Data Access)                │
│       工厂模式Fetchers + 数据处理器 + 缓存机制          │
└─────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────┐
│                   数据源层 (Data Sources)                │
│         akshare + yfinance + 本地缓存 + 配置文件        │
└─────────────────────────────────────────────────────────┘
```

## 数据流设计

### 1. 资产导入流程

```
用户输入资产代码 → API服务 → 资产管理器 → 工厂模式Fetcher → 数据源API
     ↓                ↓          ↓              ↓              ↓
UI界面接收   ←   标准化响应  ←  注册资产  ←  获取数据  ←  返回原始数据
```

### 2. 组合价值计算流程

```
获取持仓列表 → 查询资产价格 → 货币转换 → 计算各类价值 → 返回计算结果
     ↓            ↓            ↓            ↓            ↓
持仓数据     价格数据       汇率数据     计算价值     汇总展示
```

### 3. 仪表板数据更新流程

```
定时触发器 → 并行数据获取 → 数据处理 → 缓存更新 → UI刷新
     ↓           ↓           ↓         ↓         ↓
定时任务     资产数据    计算指标   更新缓存   页面更新
```

## 错误处理机制

### 多层错误处理策略

1. **UI层错误处理**: 用户友好的错误提示
2. **API层错误处理**: 标准化错误响应
3. **业务层错误处理**: 业务逻辑异常处理
4. **数据层错误处理**: 数据获取和存储异常处理

### 错误恢复机制

- **重试机制**: 网络请求失败自动重试
- **降级策略**: 主要数据源失败时使用备用数据源
- **缓存回退**: 实时数据获取失败时使用缓存数据
- **默认值处理**: 计算失败时返回安全默认值

## 设计原则

1. **开闭原则**: 系统对扩展开放，对修改封闭
2. **单一职责**: 每个模块只负责一个功能领域
3. **接口隔离**: 客户端不应依赖不需要的接口
4. **依赖倒置**: 高层模块不依赖低层模块，都依赖抽象
5. **里氏替换**: 子类可以替换父类而不影响程序功能
