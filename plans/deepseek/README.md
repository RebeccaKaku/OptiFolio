# DeepSeek 任务执行契约

本目录把 OptiFolio 的产品路线拆成可以由低成本编码模型独立完成、由 Codex 逐项审查的小任务。任务必须按编号和依赖顺序执行；一次会话只领取一个任务。

## 开工前

1. 阅读本文件、指定任务文件、`CLAUDE.md`、`docs/PRODUCT_VISION_AND_EXECUTION_PLAN.md`。
2. 执行 `git status --short --branch`，不得覆盖或清理既有未提交修改。
3. “允许修改”只限制写入；可只读检查直接依赖模块及其测试以确认真实接口。若必须写入清单外文件，先停止并汇报原因。
4. 复述输入、输出、金融语义、非目标和拟修改文件，再开始编码。

## 通用工程规则

- 路由只做 HTTP 解析与状态码映射；service 组织用例；repository/database 负责持久化；analytics 只做纯计算。
- 新接口使用 `src/services/response.py` 的统一响应结构；不得改变既有接口响应。
- API 与 analytics 金额使用 Decimal/十进制字符串或明确的整数最小单位。既有账本 SQLite REAL/float 是受控 legacy storage boundary：本阶段不顺手迁移，进入计算层时立即用 `Decimal(str(value))`，写回前做范围校验，并用明确容差测试往返。
- 日期使用 ISO `YYYY-MM-DD`；时间戳带时区；币种使用大写 ISO 4217 三字母代码。
- `None/unknown` 与 `0` 严格区分。缺价格、汇率、费用或条款时，不得填 0、1 或“低风险”。
- ID 由调用方提供或使用 UUID；测试不得依赖随机 ID 的具体值。
- 每次 schema 修改只能增加一个版本，必须提供从上一真实版本迁移、全新建库、重复初始化和失败回滚测试。
- 不读取或提交 `local/`、真实截图、数据库、账号全号、客户号、密码、密钥或真实金额。
- 不安装新框架，不做顺手重构，不修改 `FinData/`，除非任务文件明确授权。
- 不执行 `git add`、`commit`、`push`，不删除用户文件。

## 通用金融规则

- 外部出入金不是投资收益；内部转账和换汇不改变个人总财富。
- 产品计价币种、账户基础币种和报告币种是三件不同的事。
- 人工报告市值、公开净值推算值、沿用旧值必须标注不同来源和质量。
- 部分账户覆盖不得冒充完整组合；不完整时不得显示精确“今日收益”。
- 家庭未知资产不纳入个人总资产；家庭实物黄金只能作为可选的范围外备注。
- AI、宏观观点和研究模型只能生成情景输入，不能自动下单或覆盖人工确认数据。

## 最低测试要求

每项任务至少覆盖：正常路径、边界值、缺失数据、非法输入、重复操作、跨币种或日期边界（若适用）、以及一个明确的金融反例。先运行任务专项测试，再运行：

```powershell
$env:NUMBA_DISABLE_JIT='1'
python -m pytest -q --basetemp .pytest_tmp_full -p no:cacheprovider
```

不得提交 `.pytest_tmp*`、`.coverage`、数据库或缓存文件。

若验收标准相互冲突、真实代码与 spec 不符、或需要修改禁止文件，立即停止，不自行扩大范围。
